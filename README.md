[Макет "Modsen Twitter"](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=0-1&p=f&t=2GFkwJwcKDauilZ5-0)

## Необходимый функционал:

- Авторизация и регистрация пользователя(+валидация введенных данных);
- Просмотр постов пользователя и других пользователей;
- Поиск постов и пользователей;
- Возможность создания нового поста;
- Изменение личных данных пользователя;
- Подсписка на других пользователей;
- Просмотр профиля другого пользователя;
- Возможность ставить лайки под постами;
- Удаление поста;
- Infinity scroll для подгрузки постов;
- Смена темы приложения.

### Также проект предполагает:

- Организацию файловой структуры react приложения. Ссылка на структуру: [Cтруктура проекта](https://github.com/mkrivel/structure). Или [Feature sliced design](https://feature-sliced.github.io/documentation/);
- Придерживаться требований по написанию и организации кода react приложения. Ссылка на требования: [Требования к тестовому заданию](https://github.com/annaprystavka/requirements);
- Деплой приложения на любые хостинговые сервисы (Netlify, vercel, и др.);
- Настроить конфигурации **_babel_**, **_eslint_**, **_prettier_**, **_husky_**, **_commitlint_**;
- Использование TypeScript для типизирования и уменьшения количества потенциальных багов;
- Использование алиасов для импортирования файлов;
- Оптимизацию дизайна под мобильные устройства (до 390px);
- Написание технической документации в readme.md для описания работы с firebase;
- Покрытие тестами всего приложения (e2e, unit);
- Также реализация общей темы для стилей в проекте(пример общей темы, в тестовом задании используйте для стилей рекомендованную библиотеку) - [ссылка на пример](https://habr.com/ru/articles/591381/);
- Анимацию при наведения, нажатии на кнопки, прокрутки карусели и слайдеров, появлении элементов на странице при рендере и скролле;
- Хранение данных о пользователях и постах в firebase(использование Realtime Datebase);
- Реализация Loader для отображения запасного UI при подгрузке данных;
- Использование модальных окон для подтверждения действий пользователя;
- Запрет на использование сторонних библиотек для стилей.

## Описание экранов

1. Экран [Sign Up main](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=1-3094&t=2GFkwJwcKDauilZ5-0)

Пользователь первоначально должен попадать на эту страницу. Здесь он может первоначально зарегистрироваться через Google или нажав на кнопку "Sign up with email" он попадает на страницу [регистрации](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=1-1498&t=2GFkwJwcKDauilZ5-0) по почте и паролю.

2. Экран [Sign Up](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=1-1498&t=2GFkwJwcKDauilZ5-0)(Страница Sign Up)

Для того чтобы зарегистрироваться, пользователю необходимо заполнить все поля формы: свое имя, номер телефона, почту, дату рождения(возраст пользователя должен быть не менее 16 лет, выбор даты должен происходить корректно, учитывая високосные года) и пароль дважды для его подтверждения правильности ввода пароля(Все поля, в которых происходит ввод пароля должны иметь возможность скрыть и показать пароль). Все поля должны быть обязательными.  
Валидация:

- имя должно быть минимум 3 символа и максимум 20;
- при вводе телефона должна срабатывать маска ввода(+375(XX)-XXX-XX-XX) и учитывать правильно введенные коды (29, 33, 44) и длина телефона.
- почта должна работать с разными доменами;
- пароль должен быть минимум 8 символов и максимум 15, содержать цифры, латинские буквы нижнего и верхнего регистра, а также специальные символы.

В случае, если при регистрации пользователь ввел уже существующую почту, то на экране должен отобразиться в [уведомлении](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=1-3081&t=2GFkwJwcKDauilZ5-0) с этой информацией. Ошибки, связанные с запросами на уровне firebase должны быть обработаны и информация о них должна выводиться в корректном формате для пользователя.

При нажатии на "Use Google" пользователь переходит на [главную страницу регистрации](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=1-3094&t=2GFkwJwcKDauilZ5-0).

2. Экран [Log In](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=1-2230&t=2GFkwJwcKDauilZ5-0)(Страница Log In)

На данной странице пользователь может зайти в аккаунт введя свое имя и почту(валидация полей остается той же, как и на странице регистрации). В случае того, если аккаунта не существует или возникли другие ошибки при отправке данных на сервер, то необходимо оповестить об этом пользователя в [уведомлении](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=1-3081&t=2GFkwJwcKDauilZ5-0). В ином случае должно отображаться окно с успешным входом в аккаунт и пользователь переходит на [главную страницу приложения](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=1-1048&t=2GFkwJwcKDauilZ5-0).
При нажатии на "Sign up to Twitter" пользователь переходит на страницу [регистрации](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=1-1498&t=2GFkwJwcKDauilZ5-0).

3. Экран [Profile](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=1-657&t=2GFkwJwcKDauilZ5-0)(Страница Profile)

Левый сайдбар отвечает за навигацию, добавление нового твита и выход из аккаунта.
На все страницы, что не описаны в тестовом задании должны быть заглушки. Перезагрузка страницы должна работать успешно. В случае, если пользователь ввел некорректный URL, должна отображаться страница Not Found в не зависимости зарегистрирован пользователь или нет. При возврате на предыдущую страницу пользователь попадает на валидную последнюю открытую страницу.

При клике на кнопку "New Tweet" должно открываться модальное окно, в котором пользователь для нового поста может ввести текст и добавить картинку по необходимости. В случае, если пользователь пытается ввести более 500 символов, должно появляться сообщение о том, что превышен лимит. При этом ввод символов должен блокироваться, если пользователь ввел максимальное количество символов, кнопка по создаю поста должна быть задизейблена. Также под полем нужно показывать счётчик символов, например: «250 / 500». При этом валидация должна срабатывать на onChange. Максимальное разрешение картинки - 15 МБ, ожидаемые расширения - .webp, .jpg, jpeg, .png. Количество добавляемых изображений в посте - 5.

При клике на кнопку "Log out" для подтверждения действий пользователя должно показываться [модальное окно](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=12-2311&t=2GFkwJwcKDauilZ5-0), после подтверждения пользователь переходит на главную страницу входа в приложение.

В основной части приложения отображается информация о пользователе - его аватарка, баннер, имя, дополнительная информация(если указана), telegram(если указан), количество подписавшихся и на кого подписался пользователь, также кнопка редактирования личных данных. При клике на эту кнопку открывается [модальное окно](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=1-838&t=2GFkwJwcKDauilZ5-0), в котором можно поменять свое имя, данные телеграма, личную информацию и свой пол(автоматически у каждого пользвателя стоит мужской пол), всего предусматривается два пола(женский и мужской).
Есть возможность поменять аватар пользователя и баннер. Если ничего в форме не поменялось, то кнопка сохранения не должна срабатывать.

При нажатии на "Would you like to change your password?" открывается окно [смены пароля](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=1-1205&t=2GFkwJwcKDauilZ5-0). Если ничего не было изменено или валидация не прошла - кнопка сохранения не активна. Для изменения пароля пользователю нужно ввести текущий пароль и дважды написать новый. Валидация для полей должна сохраняться, как и на странице регистрации. Если пароли не совпадают, то должна сработать валидация. При успешном изменении пароля должно высветить на 5 секунд окно с успешным подтверждением(при этом модальное окно по изменению пароля закрывается и пользователь видит страницу профиля), в ином случае - информация с ошибкой(также закрывается модальное окно).

Пользователь на странице может добавить новый пост: добавить изображение и текст. Валидация для текста и изображения такая же как и в модальном окно создания нового поста.
На текущей странице пользователь видит только свои посты, на каждом посте количество лайков, если он поставил под постом лайк, то цвет изображения сердца становится красным. Есть возможность удалить свой пост, при этом должно появиться модальное окно с подтверждением действий пользователя. Во всем приложении пользователь может удалять только свои посты на любой странице.

В правом сайдбаре на странице есть возможность найти посты других пользователей. Поиск должен происходить по полному совпадению (т.к. firebase не предоставляет возможность частичного совпадения)(если Вы хотите сделать поиск по включению, то можете использовать [дополнительные возможности](https://cloud.google.com/firestore/docs/solutions/search)). Также для поиска используйте debounce. Количество найденных постов может быть максимум 10, при этом в видимой части приложения отображается три поста, а чтобы просмотреть и другие - используется скролл. Когда происходит загрузка данных, должен отображаться лоадер, он должен отображаться вместо всего блока с 6-ю представленными изображениями. В случае успешного результата вместо лоадера должен отображаться блок с найденными постами. Если в поле ввода ничего не введено, то опять должен отображаться блок с 6-ю изображениями.

При клике на нужный найденный пост открывается страница, где отображается все содержимое поста - пользователь, время создания поста, текст и/или изображение, количество лайков. Пользователь может поставить лайк. В правом сайдбаре также остается возможность поиска других постов и вместо блока с найденными постами отображается дополнительный дефолтный блок с картинками, который также отображается на странице профиля, когда поиска по постам не происходит. Также предусмотрена кнопка назад, при этом пользователь попадает на странице профиля.

В блоке "You might like" отображен список пользователей, который является рекомендуемыми, при раскрытии всего блока - максимальное количество пользователей - 10. При клике на кнопку "Follow" пользователь может подписаться на другого пользователя, при этом на странице текущего пользователя количество людей на которые подписан пользователь - увеличивается, как и на странице пользователь на которого произошла подписка - увеличивается количество подписчиков. (отображение списка пользователей подписчиков можно проработать самостоятельно, но пока что не подразумевается в текущем тестовом задании). При перезагрузке страницы рекомендуемые пользователи могут меняться, а могут остаться теми же(этот момент можно продумать самостоятельно, опираясь на то, будут ли отправлен повторно запрос, также если пользователь подписался на другого пользователя, то в дальнейшем при перезагрузке страницы этого пользователя в списке рекомендуемых он больше не отображается).

4. Экран [Home](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=1-1048&t=2GFkwJwcKDauilZ5-0)(Страница Home)

На эту страницу пользователь попадает только после первого входа(при регистрации) или в дальнейшем при авторизации. Также после этого пользователь не может вернуться назад на страницу регистрации или логина, если воспользуется возможностью перехода на предыдущую страницу. На странице есть возможность выхода пользователя из аккаунта, при этом должно отображаться [модальное окно](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=12-2311&t=2GFkwJwcKDauilZ5-0) с подтверждением действий пользователя.

В шапке страницы есть возможность сменить общую тему приложения(светлая/темная).
На странице есть возможность создать новый пост, который добавится на текущей странице и на странице самого пользователя, а также отображаются посты других пользователей. Сами посты должны подгружаться по 5 постов, чтобы сильно не подвисал пользовательский интерфейс из-за большого количества постов.
В поиске "Search Users" происходит поиск пользователей(он должен происходить на стороне firebase). Похожая логика происходит, как и с блоком поиска постов, также нужно предусмотреть лоадер и скролл. При клике на найденного пользователя, мы можем перейти на [его страницу](https://www.figma.com/design/exmlYOmXnmKnDS1AXS0kSz/Modsen-Twitter-Clone?node-id=1-1242&t=ZGvtjkHtsZSR13m5-0). Она очень схожа со станицей профиля, только нет возможности редактировать профиль, но есть подписка на пользователя. При клике на кнопку назад, пользователь попадает на страницу Home.

## Используемые технологии

### Для react

- **_node.js_** - программная платформа, основанная на движке V8 (транслирующем JavaScript в машинный код);
- **_next.js_** - фреймворк на JavaScript, использующий React для построения server-side render;
- **_babel_** - транспайлер, преобразующий код из одного стандарта в другой;
- **_eslint_** - линтер для JavaScript кода;
- **_prettier_** - инструмент для автоформатирования кода;
- **_husky_** - современное решение для управления Git хуками;
- **_commitlint_** - инструмент, который проверяет сообщения commit-ов на соответствие общепринятым стандартам их описаний;
- **_firebase_** - платформа для разработки приложений, предоставляет облачное хранилище, аналитику и многое другое;
- **_yarn_** - менеджер пакетов;
- **_react-hook-forms_** - библиотека для обработки элементов ввода формы;
- **_SCSS_** - расширение синтаксиса CSS;
- **_typescript_** - строго типизированный язык для уменьшения количества потенциальных багов;
- **_playwright_** — e2e тестирование для web-приложений;
- **_jest_** — фреймворк для unit-тестирования.

## Тестирование

Реализовать unit и e2e тестирование c полным покрытием функционала приложения:

- Модуль авторизации;
- Модуль регистрации;
- Модуль смены темы;
- Модуль поиска пользователей и твитов;
- Модуль создания нового твита;
- Модуль данных о пользователе.
